ShadowRoot Module

A custom element should not break styles and scripts the host document, and vice versa. We need a kind of encapsulation for web components: The shadowroot. It is a special kind of node that acts as an access barrier for CSS and scripts, in both directions. A CSS or script selector cannot access HTML elements on the other side of that barrier, neither below nor above. So the scope of style rules is limited to the the DOM tree they where live in: Global CSS cannot style elements below a shadowroot, and CSS below a shadowroot cannot style elements outside that shadow tree. Similarly, elemsents defined in a shadow tree are not visible outside the boundaries of this shadow tree. Such encapsulation is an essential aspect of re-usable web components.

A shadow tree is a DOM tree where the root node is a shadow root node.

A shadowroot node can be created programmatically or declarativly. Here is an example of programmatic creation:

<div id="shadowrootHost"></div>
<script type="module">
shadowrootHost.attachShadow({mode:"open"});
const p = document.createElement( "P" );
p.textContent = "I'm in the shadowtree" ;
shadowrootHost.shadowRoot.append( p );
</script>

And here the declarative approach:

<div id="shadowrootHost">
	<template shadowrootmode="open">
		<p>I'm in the shadowtree</p>
	</template>
</div>

Crossing the Border

The access-barrier, created by a shadow root node, is not impenetrable for JavaScript if the shadow root was created with shadowrootmode="open":

shadowRootHost.shadowRoot.querySelectorAll( "P" )

Elements can be accessed via the shadowRoot property of the shadow root host.

Shadow Parts Module

The CSS Shadow Parts module adresses the need for global CSS to style elements across a shadowroot boundary. A web component author can decide to make certain elements below the shadowroot accessible to global CSS rules.

part attribute

The partattribute on a shadowroot host element contains a comma separated list of elements in the shadowroot. These elements are made accessible to the CSS :part() pseudo element selector in the parent DOM tree (where the host element lives in). The visibility stretches across exactly one shadowroot boundary. So if a web component is nested in another web component, the outer web component can style the published child component elements, but the global CSS in the root document cannot.

exportparts attribute

The exportparts attribute on a shadowroot host element works similiar to the part attribute, but the accessibility is extended to the entire document above the shadow root.

The attribute contains a comma-separated list of id:alias mappings, where the id references an element in the shadow tree beneath the shadow root, and the alias is the name under which it is published to the :part() CSS selector.

:part() CSS pseudo-element

The :part() pseudo-element in a stylescheed selects an element in a shadow DOM tree that has been made accessible through the part or partlist attributes on shadowroot host elements.
